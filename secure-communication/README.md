# Secure Soldier-to-Soldier Communication Network

A beginner-friendly cybersecurity demonstration project that simulates secure military communication over a local area network (LAN). This system demonstrates fundamental security concepts including encryption, authentication, replay attack prevention, attack detection, and threat monitoring.

## 🎯 Project Overview

This project uses **3 laptops** running the **same codebase** in different roles:
- **Laptop A (Sender)**: Encrypts and sends secure messages
- **Laptop B (Receiver)**: Receives, validates, and decrypts messages
- **Laptop C (Attacker)**: Intercepts messages and attempts replay attacks

The system demonstrates:
- ✅ **Encryption**: Fernet (AES-128-CBC + HMAC-SHA256)
- ✅ **Authentication**: HMAC-SHA256 for sender verification
- ✅ **Replay Attack Prevention**: Timestamp + sequence number validation
- ✅ **Attack Detection**: Real-time logging of security violations
- ✅ **Threat Monitoring**: Dynamic threat level indicators (SAFE/WARNING/ATTACK)

## 🚀 Quick Start

### Prerequisites
- Python 3.8 or higher
- 3 laptops on the same WiFi network
- pip (Python package manager)

### Installation

1. **Clone or download this repository** to all 3 laptops

2. **Install dependencies** on all laptops:
   ```bash
   cd secure-communication
   pip install -r requirements.txt
   ```

3. **Generate keys** (on one laptop):
   ```bash
   python setup.py
   ```

4. **Copy the entire folder** to all 3 laptops (ensures same keys and code)

5. **Configure IP addresses** in `config.json`:
   - Find Laptop B's IP address: `ipconfig` (Windows) or `ifconfig` (Mac/Linux)
   - Update `receiver_ip` in `config.json` on all laptops

6. **Run the demonstration**:
   - **Laptop B**: `python receiver.py`
   - **Laptop C**: `python attacker.py`
   - **Laptop A**: `python sender.py`

For detailed instructions, see [SETUP.md](SETUP.md).

## 📖 Module Documentation

### config.py
Manages configuration loading and cryptographic key generation.

**Key Functions**:
- `generate_keys()` - Generates Fernet encryption key and HMAC authentication key
- `load_config(config_path)` - Loads configuration from JSON file
- `get_network_config()` - Returns network parameters (IP, port, timeout)
- `get_security_config()` - Returns security parameters (keys, timestamp window)
- `get_display_config()` - Returns display/color configuration

### crypto_utils.py
Implements cryptographic operations for encryption and authentication.

**Key Functions**:
- `encrypt_message(plaintext, key)` - Encrypts message using Fernet
- `decrypt_message(ciphertext, key)` - Decrypts message using Fernet
- `generate_hmac(message, key)` - Generates HMAC-SHA256 authentication token
- `verify_hmac(message, hmac_digest, key)` - Verifies HMAC authentication token

**Security Features**:
- Fernet encryption (AES-128-CBC + HMAC-SHA256)
- Constant-time HMAC comparison (prevents timing attacks)
- UTF-8 encoding for string inputs

### network_utils.py
Handles TCP socket communication for sending and receiving messages.

**Key Functions**:
- `create_server_socket(host, port)` - Creates and binds TCP server socket
- `accept_connection(server_socket)` - Accepts incoming client connection
- `create_client_socket(host, port, timeout)` - Creates and connects TCP client socket
- `send_message(sock, message_dict)` - Sends JSON message with length prefix
- `receive_message(sock, buffer_size)` - Receives JSON message with length prefix

**Features**:
- 4-byte length prefix for reliable message framing
- JSON serialization for structured data
- Error handling for connection failures

### logger.py
Tracks security violations and calculates threat levels.

**Key Classes**:
- `ThreatLevel` - Enum with SAFE, WARNING, ATTACK values
- `AttackLog` - Data class for attack log entries

**Key Functions**:
- `log_attack(attack_type, details)` - Logs a security violation
- `get_attack_count()` - Returns total number of detected attacks
- `get_threat_level()` - Calculates current threat level based on attack count
- `display_attack_log(colors)` - Displays all logged attacks with timestamps
- `display_threat_level(colors)` - Displays current threat level with ANSI colors

**Threat Levels**:
- **SAFE** (Green): 0 attacks
- **WARNING** (Yellow): 1-2 attacks
- **ATTACK** (Red): 3+ attacks

### sender.py
Sender role implementation (Soldier A).

**Main Flow**:
1. Load configuration and cryptographic keys
2. Connect to receiver via TCP
3. Accept user input for messages
4. For each message:
   - Encrypt using Fernet
   - Generate HMAC authentication token
   - Attach timestamp and sequence number
   - Send as JSON message
5. Display confirmation with color coding

**Message Format**:
```json
{
  "encrypted_payload": "base64-encoded-ciphertext",
  "hmac": "hex-encoded-hmac-digest",
  "timestamp": 1705329135.234,
  "sequence_number": 42
}
```

### receiver.py
Receiver role implementation (Soldier B).

**Main Flow**:
1. Load configuration and cryptographic keys
2. Create server socket and listen for connections
3. Accept connection from sender
4. For each received message:
   - Validate HMAC authentication
   - Check timestamp freshness (within 60 seconds)
   - Validate sequence number (must be greater than last)
   - Decrypt message if valid
   - Log attack if invalid
   - Update and display threat level

**Validation Logic**:
- **HMAC Verification**: Rejects messages with invalid authentication
- **Timestamp Validation**: Rejects messages older than 60 seconds or from future
- **Sequence Number Validation**: Rejects messages with non-increasing sequence numbers

### attacker.py
Attacker role implementation (Hacker).

**Capabilities**:
- Capture messages (manual entry for demo)
- View captured messages (shows encrypted data is unreadable)
- Replay captured messages to receiver

**Limitations** (by design):
- No encryption key (cannot decrypt messages)
- No authentication key (cannot forge valid HMAC)
- Cannot create fresh messages that pass validation

**Menu Options**:
1. Capture message (manual entry)
2. View captured messages
3. Replay captured message
4. Exit

## 🔒 Security Features

### Encryption
- **Algorithm**: Fernet (AES-128-CBC + HMAC-SHA256)
- **Key Size**: 256 bits (32 bytes)
- **Purpose**: Ensures confidentiality - attackers cannot read intercepted messages

### Authentication
- **Algorithm**: HMAC-SHA256
- **Key Size**: 256 bits (32 bytes)
- **Purpose**: Verifies sender identity - prevents message forgery

### Replay Attack Prevention
- **Timestamp Validation**: Rejects messages older than 60 seconds
- **Sequence Number Validation**: Rejects messages with non-increasing sequence numbers
- **Purpose**: Prevents attackers from reusing captured messages

### Attack Detection
- **Real-time Logging**: All security violations are logged with timestamps
- **Threat Level Monitoring**: Dynamic indicators (SAFE/WARNING/ATTACK)
- **Attack Types Detected**:
  - FAKE_SENDER: Invalid HMAC authentication
  - REPLAY_ATTACK: Old timestamp or duplicate sequence number

## 🎓 Educational Value

This project demonstrates:
- **Symmetric Encryption**: How encryption protects data confidentiality
- **Message Authentication**: How HMAC prevents message forgery
- **Replay Attack Prevention**: How timestamps and sequence numbers prevent message reuse
- **Threat Monitoring**: How security systems detect and respond to attacks
- **Network Security**: How secure communication works over untrusted networks

## 🛠️ Configuration

Edit `config.json` to customize:

```json
{
  "network": {
    "receiver_ip": "192.168.1.100",  // Receiver's IP address
    "receiver_port": 5555,            // Port for communication
    "buffer_size": 4096,              // Socket buffer size
    "timeout": 30                     // Connection timeout (seconds)
  },
  "security": {
    "encryption_key": "...",          // Fernet encryption key (generated)
    "auth_key": "...",                // HMAC authentication key (generated)
    "timestamp_window": 60,           // Max age for timestamps (seconds)
    "replay_window": 300              // Sequence tracking window (seconds)
  },
  "display": {
    "colors": {
      "safe": "\u001b[92m",          // Green
      "warning": "\u001b[93m",        // Yellow
      "attack": "\u001b[91m",         // Red
      "reset": "\u001b[0m",           // Reset
      "info": "\u001b[94m",           // Blue
      "attacker": "\u001b[95m"        // Magenta
    }
  }
}
```

## 🐛 Troubleshooting

Common issues:
- **Connection refused**: Ensure receiver is running first, check IP addresses
- **Port already in use**: Change port in config.json or stop existing process
- **Key errors**: Run `python setup.py` to regenerate keys
- **No colors**: Use a terminal that supports ANSI colors (Windows Terminal, iTerm2, etc.)

For detailed troubleshooting, see [TROUBLESHOOTING.md](TROUBLESHOOTING.md).
